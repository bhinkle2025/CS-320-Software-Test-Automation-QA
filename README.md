# CS-320-Software-Test-Automation-QA

I ensure my code, program, and software are functional and secure by incremental testing, input validation, and careful review. Instead of waiting until the end of development to test everything, I break the program into smaller components and test each part as I build. This helps me catch errors early and confirm that each requirement works as intended. I also make sure that invalid input is rejected, that reasonably valid input is accepted, and that the program handles errors without crashing or behaving unpredictably. 

I interpret user needs by breaking them down into clear, manageable requirements. Once I understand what the user expects the program to do, I can translate those needs into specific features, conditions, and tests. This helps ensure the final program is based on the user's goals rather than my assumptions. 

My approach to software design is to look at both the larger system and the smaller details. At the macro level, I consider how the whole program should function and how the main parts should work together. At the micro level, I focus on individual methods, inputs, outputs, and edge cases. This allows me to design software that is organized, testable, functional, and secure.
